# План розробки Botpilot

## 1. Архітектура та бекенд (Пріоритет)
- [ ] Переписати серверний проксі-механізм (`/api/proxy-auth`), щоб він динамічно визначав біржу (Gate.io, Binance, OKX) за вказаним URL або параметром та застосовував відповідний алгоритм підпису (HMAC-SHA512 для Gate, HMAC-SHA256 для Binance та OKX).
- [ ] Винести бізнес-логіку підписів у модульну структуру без жорсткого хардкоду під одну біржу.

## 2. Управління біржами та балансом
- [ ] Реалізувати підтримку підключення кількох бірж (Gate.io, Binance, OKX) із перевіркою коректності ключів та прав доступу (Spot / Futures).
- [ ] Додати автоматичне оновлення балансу для активних бірж.

## 3. Торгові дані та ринкові інструменти (Public API)
- [ ] Налаштувати універсальне отримання ціни та свічок (Ticker / Klines) для вибраної пари.
- [ ] Підтягування специфікацій контрактів (розмір лота, крок ціни) залежно від обраної біржі.

## 4. Ф'ючерсний грід-бот (Trade API)
- [ ] Розміщення сітки лімітних ордерів на купівлю та продаж.
- [ ] Отримання та відображення списку активних ордерів (`open orders`).
- [ ] Моніторинг відкритих позицій та розрахунок поточного PnL.
- [ ] Логіка скасування ордерів та зупинки бота.
